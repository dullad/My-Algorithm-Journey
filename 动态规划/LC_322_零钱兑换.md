# LeetCode 322. 零钱兑换

### 1. 题目链接
[https://leetcode.cn/problems/coin-change/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
dp[i]，组成数量i的最小硬币个数，初始化为INT_MAX。
*【思路梳理】：思路正确，定义一维数组寻找最小步数路径。*
**易错点：** 在进行 `dp[i] = min(dp[i], dp[i-coins[j]] + 1)` 状态更新时，未对前置状态的合法性做有效排查。如果使用了值为 `INT_MAX` 的无效位置并作 $+1$ 运算会导致严重的整型溢出错误。

### 3. 核心解法：动态规划（完全背包）
**3.1 思路分析**
**破局点：** 将问题视为“硬币重量为代价，价值皆为 1，求装满容量的最小价值”的背包求解，每访问到一个总金额，遍历最后一枚投入的面值，更新方程为 `dp[i] = min(dp[i], dp[i - coin] + 1)`。

**3.2 代码实现 (C++)**
```c++
// 完整写出
class Solution {
public:
    int coinChange(vector<int>& coins, int amount) {
        vector<int> dp(amount + 1, INT_MAX);
        dp[0] = 0;
        for(int i = 1; i <= amount; i++){
            for(int j = 0; j < coins.size(); j++){
                if(coins[j] <= i && dp[i - coins[j]] != INT_MAX){
                    dp[i] = min(dp[i], dp[i-coins[j]] + 1);
                }
            }
        }
        if(dp[amount] == INT_MAX) return -1;
        return dp[amount];
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(S 	imes N)$。$S$ 表示金额大小，$N$ 代表硬币种类的数量，嵌套层级明确。
- **空间复杂度**: $O(S)$。状态矩阵总长即为我们需要凑出的目标面额。

### 5. 总结与反思
完全背包求最小值的最标准定式，依赖严谨的初始大值隔离来实现不可达路径剔除。
