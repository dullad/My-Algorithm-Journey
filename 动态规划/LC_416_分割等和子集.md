# LeetCode 416. 分割等和子集

### 1. 题目链接

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
等价于是否存在一种组合，和为sum/2（sum为偶数的前提下）。但是思路不清晰。
*【思路梳理】：转换成找“和为sum/2”的目标是正确的，在此基础上，这个组合选择过程可以被直接具现化成对该指定容量大小的背包装填，属于典型的0-1背包问题判定。*
**易错点：** 若采用内层正序填表（即让小背包先更新），必然导致同一物品被利用在后续更新里，这会退化为不限选取次数的“完全背包”。零一背包中需要内层降序扫描。

### 3. 核心解法：动态规划（0-1背包降维）
**3.1 思路分析**
**破局点：** 将判定对半分化剥换为：给定可用承重 $W = 	ext{sum} / 2$，每一个数字为一枚对应重量价值双重的金块，求是否存在恰好达到此容载极值的方式。用一维状态布尔向量自大向小压缩判定。

**3.2 代码实现 (C++)**
```c++ 动态规划
// 看题解复现
class Solution {
public:
    bool canPartition(vector<int>& nums) {
        int sum = 0;
        for(int i = 0; i < nums.size(); i++){
            sum += nums[i];
        }
        if(sum % 2 != 0) return false;
        sum /= 2;
        vector<bool> dp(sum + 1, false);// 是否存在和为i的子集
        dp[0] = true;
        for(int i = 0; i < nums.size(); i++){
            for(int j = sum; j >= nums[i]; j--){
                dp[j] = dp[j] || dp[j - nums[i]];
            }
        }
        return dp[sum];
    }
};
```

```c++ 记忆化搜索dfs
class Solution {
    int dfs(vector<int>& nums, int endI, int sum, vector<vector<int>>& memo){
        // [0,endI]中是否存在和为sum的子集
        if(sum == 0) return 1;
        if(endI < 0) return 0;
        // 记忆查找，避免复杂度爆炸
        if(memo[endI][sum] != -1) return memo[endI][sum];
        if(sum < nums[endI]) return dfs(nums, endI - 1, sum, memo);
        else{
            memo[endI][sum] = dfs(nums, endI - 1, sum, memo) || dfs(nums, endI - 1, sum - nums[endI], memo);
        }
        return memo[endI][sum];
    }
public:
    bool canPartition(vector<int>& nums) {
        int sum = 0;
        for(int i = 0; i < nums.size(); i++){
            sum += nums[i];
        }
        if(sum % 2 != 0) return false;
        sum /= 2;
        vector<vector<int>> memo(nums.size(), vector<int>(sum + 1, -1));
        return dfs(nums, nums.size() - 1, sum, memo);
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N 	imes 	ext{sum})$。需要填补行宽为集合一半、深为全体集数的完整面。
- **空间复杂度**: 经过精炼化只有 $O(	ext{sum}/2)$ 容量长度消耗的一维横条。

### 5. 总结与反思
利用背包模板进行存在性验证，是解构并离散数学凑等分策略的最佳实践。
