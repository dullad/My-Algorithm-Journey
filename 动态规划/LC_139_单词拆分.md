# LeetCode 139. 单词拆分

### 1. 题目链接
[https://leetcode.cn/problems/word-break/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
不会。
*【思路梳理】：此题误区在于很多人试图从字典里的词去“拼凑”或“贪心覆盖”长字符串，一旦前缀出现交叠词会导致回溯失败。解决途径应该是倒置探勘主体：直接验证长串的逐级前缀能否合法解构。*
**易错点：** 贪心匹配无法兼顾多种重叠子串的有效性（如 `s="applepenapple"`, `dict=["apple", "pen"]`），遇到存在相同前缀选项时将无法继续，需要记录历史判定状态递推。

### 3. 核心解法：动态规划
**3.1 思路分析**
**破局点：** 将“原字符串合法性”转移为其“前段合法性 AND 后侧子串存在于字典”的双验证机制。设置布尔状态数组 `dp[i]` 标识长串前 `i` 位已被成功分配。然后对每个位置 `i` ，寻找是否存在分割点 `j` 使得 `dp[j]` 且剩余串 `str[j:i-1]` 能在哈希集合中命中。

**3.2 代码实现 (C++)**
```c++
// 看题解复现
class Solution {
public:
    bool wordBreak(string s, vector<string>& wordDict) {
        unordered_set<string> myset(wordDict.begin(), wordDict.end());
        vector<bool> dp(s.length() + 1, false);// 前i个字符能否被成功拆分
        dp[0] = true;
        for(int i = 1; i <= s.length(); i++){
            for(int j = 0; j < i; j++){// 前j个字符能否被拆分,剩下字符是否在set中
                if(dp[j] && myset.find(s.substr(j, i - j)) != myset.end()){
                    dp[i] = true;
                    break;
                }
            }
        }
        return dp[s.length()];
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N^2)$。两层迭代长串规模为 $O(N^2)$，如果考虑子串提取和哈希开销最坏还会乘以串长因子。
- **空间复杂度**: $O(N)$。用于查全匹配的一维数组及哈希表开销，与单词词汇映射成比例。

### 5. 总结与反思
利用无后效性逐波确认字符串的前缀完备性，结合哈希快速判定子区间，是求解模糊组装的最佳路径。
