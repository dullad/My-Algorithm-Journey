# LeetCode 300. 最长递增子序列

### 1. 题目链接
[https://leetcode.cn/problems/longest-increasing-subsequence/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
dp[i]：以nums[i]为序列结尾的最长递增序列长度。每个元素回头遍历找比它之前小的元素，检查拼上以后是否变长，更新dp[i]。
*【思路梳理】：动态规划思路完全正确。*
**易错点：** 若不小心将 `dp[i]` 直接定义为全局长 $i$ 的范围极值，会导致状态不能与新增的数字进行接续比对。递推务必绑定必须包含当前项。

### 3. 核心解法：动态规划 / 贪心二分
**3.1 思路分析**
**破局点：** 在经典的 `dp[i] = max(dp[i], dp[j] + 1)` 递推思路之外，还存在极度优化的“纸牌叠放”模型：始终维护一个“潜力最好”（增长最慢最紧凑）的长递增队列，每来一个数替换队列中首个大于它的数或延长接龙。

**3.2 代码实现 (C++)**
```c++ 动态规划
// 完整写出
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        int n = nums.size();
        // dp[i] 以nums[i]为序列结尾的最长递增序列长度
        vector<int> dp(nums.size(), 1);
        int ans = 1;
        for(int i = 0; i < n; i++){
            // 每个元素，回过头找起之前比它小的元素，可以接到后面拼接一个更长的序列
            for(int j = 0; j < i; j++){
                if(nums[i] > nums[j]){
                    dp[i] = max(dp[j] + 1, dp[i]);
                }
            }
            ans = max(dp[i], ans);
        }
        return ans;
    }; 
};
```

```c++ 贪心二分优化
// 看题解复现
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        vector<int> tails;// 长度为i+1的最长递增子序列的最小末尾数字为tails[i]
        tails.push_back(nums[0]);
        for(int i = 1; i < nums.size(); i++){
            if(nums[i] > tails.back()){
                tails.push_back(nums[i]);
            }
            else{// 找到tails中第一个比它大的数，注意lower_bound等的用法原理
                auto it = lower_bound(tails.begin(), tails.end(), nums[i]);
                *it = nums[i];
            }
        }
        return tails.size();
    }; 
};
```

### 4. 复杂度分析
- **时间复杂度**: 动态规划为 $O(N^2)$；如果采用贪心结合二分为 $O(N \log N)$ 能够大幅度消减耗时。
- **空间复杂度**: 均开辟了最坏为 $O(N)$ 状态记录或尾记录序列辅助表。

### 5. 总结与反思
保留最有潜力的紧凑状态以换取对接空间的宽容度，是这道 LIS 经典母题走向对数复杂度的金钥匙。
