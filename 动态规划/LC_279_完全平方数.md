# LeetCode 279. 完全平方数

### 1. 题目链接
[https://leetcode.cn/problems/perfect-squares/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
dp[i]：和为i的完全平方数的最小数量，需要初始化为n+1,INT_MAX。对于每个i，需要遍历小于它的所有平方数，比较包含该平方数分解和不包含该平放式分解的数量大小，取小。
*【思路梳理】：思路正确，属于典型的可以将任意平方数视作无限物品的完全背包基础模型。*
**易错点：** 在内层探测循环中，条件必须是 `$j * j \le i$`，如果误使用 `<`，则会漏掉数字 $i$ 本身恰好是完全平方数的边界场景。

### 3. 核心解法：动态规划（完全背包）
**3.1 思路分析**
**破局点：** 抽象为“可以使用无数次的物品（完全平方数）填满固定容量背包（数字 $i$）的最小物品数量”问题，直接套用转移方程 `dp[i] = min(dp[i], dp[i - j * j] + 1)`。

**3.2 代码实现 (C++)**
```c++
// 不建平方数表的方法参考题解，建表的方法完整写出
class Solution {
public:
    int numSquares(int n) {
        vector<int> dp(n + 1, INT_MAX);// 组成 i 的完全平方数最小数量
        dp[0] = 0;
        for(int i = 1; i <= n; i++){
            for(int j = 1; j * j <= i; j++){ // 无需建平方数表，直接遍历j，取平方就是平方数
                dp[i] = min(dp[i], dp[i - j * j] + 1);
            }
        }
        return dp[n];
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N\sqrt{N})$。外层循环执行数字 $N$ 次，内层只需推进至 $\sqrt{i}$，综合起来推敲。
- **空间复杂度**: $O(N)$。维护了长为 $n+1$ 的最值状态记录数组。

### 5. 总结与反思
完全背包问题求极值的转化典范，利用自身就是平方项的特性节省了打表开支。
