# LeetCode 198. 打家劫舍

### 1. 题目链接
[https://leetcode.cn/problems/house-robber/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
dp[i]表示偷到第i个房屋，最多偷多少钱。取决于是否偷这一家，取大值。
*【思路梳理】：思路大体正确。具体来说，抉择的双方是“偷这家（加上前两家的最大金额）”与“不偷这家（沿用前一家的最大金额）”。*
**易错点：** 对边界状态的判漏。如果房屋总数只有 1 间或 2 间，在不作判断的情况下直接赋值给 `dp[1]` 会引发数组越界。

### 3. 核心解法：动态规划
**3.1 思路分析**
**破局点：** 将问题拆解为不相邻元素最大和的情况，确立状态转移方程 `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`，从而逐个推演至序列终点。

**3.2 代码实现 (C++)**
```c++
// 完整写出
class Solution {
public:
    int rob(vector<int>& nums) {
        vector<int> dp(nums.size(), 0);// 偷到i，最多偷多少金额
        if(nums.size() == 1) return nums[0];
        dp[0] = nums[0];
        dp[1] = max(nums[0], nums[1]);
        for(int i = 2; i < nums.size(); i++){
            dp[i] = max(dp[i-1], dp[i-2] + nums[i]);
        }
        return dp[nums.size() - 1];
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N)$。遍历一次房屋数组直接求解。
- **空间复杂度**: $O(N)$。构建了一维状态数组 `dp`（可以通过滚动变量压至 $O(1)$）。

### 5. 总结与反思
针对“相邻不可取”模型的一维线性动态规划，其基础核心依然是通过取舍推导局部最优到全局最优。
