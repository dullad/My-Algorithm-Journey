# LeetCode 118. 杨辉三角

### 1. 题目链接
[https://leetcode.cn/problems/pascals-triangle/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
第i层有i个数，最左边和最右边是1，中间的所有数都是肩膀上两个数之和。
*【思路梳理】：思路正确。*
**易错点：** 数组索引处理极易越界。在原始思路中 `i` 代表层数并从 2 开始枚举，因此在引用上一层时需要进行 `i-2` 偏移，这种非从 0 开始的对应关系容易引发混乱。

### 3. 核心解法：动态规划
**3.1 思路分析**
**破局点：** 明确状态转移方程 `dp[i][j] = dp[i-1][j-1] + dp[i-1][j]`。按照层状结构自顶向下逐层构建数组，同时将每层的首尾元素固定置为 1，只需填充中间段即可。

**3.2 代码实现 (C++)**
```c++
// 完整写出
class Solution {
public:
    vector<vector<int>> generate(int numRows) {
        vector<vector<int>> ans;
        vector<int> temp(1, 1);
        ans.push_back(temp);
        for(int i = 2; i <= numRows; i++){ // i 表示层数
            vector<int> layer(i, 1);
            for(int j = 0; j < ans[i - 2].size() - 1; j++){
                layer[j + 1] = ans[i - 2][j] + ans[i - 2][j + 1];
            }
            ans.push_back(layer);
        }
        return ans;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N^2)$。嵌套两层循环遍历所有被构建的数字。
- **空间复杂度**: $O(1)$。如果不计算返回答案结构所需占据的二维数组体积，推导过程中并无额外占用。

### 5. 总结与反思
根据二维坐标下的相对位置规律直接累加，本质上是对递推公式的直接程序化模拟。
