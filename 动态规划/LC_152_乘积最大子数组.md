# LeetCode 152. 乘积最大子数组

### 1. 题目链接
[https://leetcode.cn/problems/maximum-product-subarray/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
维护当前元素前乘积的最大值和最小值，每个元素进来以后分别乘以最大值和最小值，来更新新的最大值和最小值。此过程中维护全局最大值。维护最小值原因是因为如果是两个很大的负数相乘可能会产生很大的正数。
*【思路梳理】：思路非常清晰且准确，深刻洞察了负号乘除极值跳反的本质。*
**易错点：** 在同一轮的赋值操作中，更新极小值时如果调用已经先被更新过的新的极大值参与推算，就会导致本轮取值混乱，必须引用前置存储的纯净状态进行横向比对。

### 3. 核心解法：动态规划（双状态同步传递）
**3.1 思路分析**
**破局点：** 在单状态记录失效的情境下增加一条暗线，通过同步推算并维系**包含当前项的最大端与最小端向后延续**。利用正负翻转使极小极值互换，彻底解决跌落陷阱。

**3.2 代码实现 (C++)**
```c++
// 完整写出
class Solution {
public:
    int maxProduct(vector<int>& nums) {
        int pre_max = nums[0];
        int pre_min = nums[0];
        int ans = nums[0];
        for(int i = 1; i < nums.size(); i++){
            int temp_pre_max = pre_max;// 注意状态覆盖
            pre_max = max(nums[i], max(nums[i] * temp_pre_max, nums[i] * pre_min));
            pre_min = min(nums[i], min(nums[i] * temp_pre_max, nums[i] * pre_min));
            ans = max(pre_max, ans);
        }
        return ans;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N)$。一次直接无回头的一维数组扫描。
- **空间复杂度**: $O(1)$。仅运用固定数量常量接棒交换最值状态。

### 5. 总结与反思
极性翻转赋予了最小值成为最大值的潜力，双轨并行交相更替可跨越局部凹谷的制约。
