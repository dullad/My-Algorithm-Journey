# LeetCode 148. 排序链表

### 1. 题目链接

[LeetCode 148. 排序链表](https://leetcode.cn/problems/sort-list/description/?envType=study-plan-v2&envId=top-100-liked)

### 2. 我的初次思路与遇到的问题

初次思路是选择排序，结果超时。

问题本质：
- 选择排序时间复杂度是 $O(n^2)$，而题目要求达到 $O(n\log n)$。
- 链表不支持随机访问，不适合快排这类依赖下标的写法。

### 3. 核心解法：归并排序（分治）

**3.1 思路分析**

链表做排序，归并是最自然的方案。

整体流程：
1. 用快慢指针找中点并断开链表，分成左右两段。
2. 递归排序左右两段。
3. 合并两个有序链表。

关键细节：
- `fast = head->next` 可以让 `slow` 落在左半部分尾部，方便断开。
- `slow->next = nullptr` 不能漏，否则递归不会收敛。

**3.2 代码实现 (C++)**

```cpp
class Solution {
    ListNode* mergeTwo(ListNode* a, ListNode* b) {
        ListNode dummy(0);
        ListNode* cur = &dummy;

        while (a != nullptr && b != nullptr) {
            if (a->val < b->val) {
                cur->next = a;
                a = a->next;
            } else {
                cur->next = b;
                b = b->next;
            }
            cur = cur->next;
        }

        cur->next = (a != nullptr) ? a : b;
        return dummy.next;
    }

    ListNode* splitMid(ListNode* head) {
        ListNode* slow = head;
        ListNode* fast = head->next;

        while (fast != nullptr && fast->next != nullptr) {
            slow = slow->next;
            fast = fast->next->next;
        }

        ListNode* mid = slow->next;
        slow->next = nullptr; // 关键，需要断开，否则会出错
        return mid;
    }

public:
    ListNode* sortList(ListNode* head) {
        if (head == nullptr || head->next == nullptr) {
            return head;
        }

        ListNode* mid = splitMid(head);

        ListNode* left = sortList(head);
        ListNode* right = sortList(mid);

        return mergeTwo(left, right);
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(n\log n)$。
- **空间复杂度**: $O(\log n)$（递归栈）。

补充：如果是迭代版自底向上归并，才可以把额外空间优化到 $O(1)$。

### 5. 总结与反思

- **链表排序首选归并**
- 模板记忆：`split -> sort(left/right) -> merge`。
- 易错点复盘：中点断链一定要做，否则会死递归。
