#### 160. 相交链表 (Intersection)
**破局思路**：
- **哈希法 (Set)**：简单粗暴，先存 A 的所有节点，再遍历 B 查重。
- **双指针法 (A+B) [推荐]**：浪漫相遇。两个指针分别走 `A -> B` 和 `B -> A` 的路径。如果是相交链表，它们走过的路程总长度均是 `Len(A) + Len(B)`，必定会在“交点”处（或终点 nullptr）相遇。

**关键细节**：
- 只有当指针为 `nullptr` 时才切换到另一条链表的头，否则正常 `next`。
- 如果不相交，最后两者都会同时变成 `nullptr`，循环结束返回 `nullptr`，逻辑自洽。

**代码实现 (双指针)**
```cpp
class Solution {
public:
    ListNode *getIntersectionNode(ListNode *headA, ListNode *headB) {
        ListNode *A = headA, *B = headB;
        while(A != B){
            // 走完自己走对方
            A = A != nullptr ? A->next: headB;
            B = B != nullptr ? B->next: headA;
        }
        return A;
    }
};
```

#### 206. 反转链表 (Reverse)
**破局思路**：
- **三指针迭代**：`prev` (前驱), `cur` (当前), `next` (后继/temp)。
- 每次操作：先把 `cur->next` 存起来，然后让 `cur` 指向 `prev`，最后整体前移。

**关键细节**：
- 循环结束条件是 `cur == nullptr`。
- 最后返回的是 `prev`（新的头节点），而不是 `cur`。

**代码实现**
```cpp
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        ListNode* prev = nullptr;
        ListNode* cur = head;
        while(cur != nullptr){
            ListNode* temp = cur->next; // 1. 暂存断开点
            cur->next = prev;           // 2. 反转当前指向
            prev = cur;                 // 3. prev 前移
            cur = temp;                 // 4. cur 前移
        }
        return prev;
    }
};
```

#### 234. 回文链表 (Palindrome)
**破局思路**：
- **快慢指针 + 栈 (您的解法)**：利用快慢指针找到中点，遍历过程中将**前半部分入栈**。然后比对栈顶（前半部分逆序）与后半部分。
- **进阶优化 ($O(1)$ 空间)**：找到中点 -> **反转后半部分链表** -> 比较 -> (可选)恢复链表。

**关键细节**：
- **奇偶处理**：`fast` 指针在结束时不同。
  - 偶数长：`f->next` 还没空，前半部分刚好入栈完。
  - 奇数长：`f->next` 空了，中间那个节点不需要比对，`s` 应该再走一步跳过中间节点。
  
**代码实现 (栈方法)**
```cpp
class Solution {
public:
    bool isPalindrome(ListNode* head) {
        stack<int> my_st;
        ListNode* s = head;
        ListNode* f = head;
        // 找中点并入栈前半部分
        while(f->next != nullptr && f->next->next != nullptr){
            my_st.push(s->val);
            s = s->next;
            f = f->next->next;
        }
        // 处理偶数长度的最后一个前半段节点
        if(f->next != nullptr){
            my_st.push(s->val);
        }
        s = s->next; // 移动到后半段开始比较
        while(s != nullptr){
            if(my_st.top() != s->val) return false;
            my_st.pop();
            s = s->next;
        }
        return true;
    }
};
```

#### 141 & 142. 环形链表 (Cycle)
**破局思路**：
- **判断有环**：龟兔赛跑。`fast` 走两步，`slow` 走一步。如果有环，`fast` 必追上 `slow`。
- **找入环点**：
    1. 相遇后，让其中一个指针（如 `fast`）**回到 `head`**。
    2. `fast` 和 `slow` 同时改为**每次走一步**。
    3. 再次相遇点即为入环点 (数学证明：$a = c(n-1) + b$)。

**关键细节**：
- `while` 循环条件必须检查 `fast` 和 `fast->next` 防止空指针引用。
- 找入环点时，第二次追逐速度必须一致（都走 1 步）。

**代码实现 (找入环点)**
```cpp
class Solution {
public:
    ListNode *detectCycle(ListNode *head) {
        ListNode* fast = head;
        ListNode* slow = head;
        while(fast != nullptr && fast->next != nullptr){
            slow = slow->next;
            fast = fast->next->next; 
            if(slow == fast){ // 相遇
                fast = head;  // 重置
                while(slow != fast){
                    slow = slow->next;
                    fast = fast->next; // 同速
                }
                return fast;
            }
        }
        return nullptr;
    }
};
```

#### 21. 合并两个有序链表 (Merge)
**破局思路**：
- **虚拟头节点 (Dummy Head)**：创建一个哨兵节点 `dummy`，不仅避免处理头指针为空的尴尬，最后直接返回 `dummy->next` 即可。

**优化建议**：
- **节点复用**：原代码中使用了 `new ListNode()` 创建新节点。实际上，合并链表通常不需要开辟新空间，直接修改节点的 `next` 指针进行“拼接”即可（即 `cur->next = list1`），这样空间复杂度是 $O(1)$。

**代码实现 (拼接优化版)**
```cpp
class Solution {
public:
    ListNode* mergeTwoLists(ListNode* list1, ListNode* list2) {
        ListNode* dummyHead = new ListNode(); // 哨兵
        ListNode* cur = dummyHead;
        
        while(list1 != nullptr && list2 != nullptr){
            if(list1->val < list2->val){
                cur->next = list1; // 直接接上原节点
                list1 = list1->next;
            }else{
                cur->next = list2;
                list2 = list2->next;
            }
            cur = cur->next;
        }
        // 接上剩余部分
        cur->next = list1 != nullptr ? list1 : list2;
        return dummyHead->next;
    }
};
```

### 3. 复杂度分析

- **时间复杂度**: 所有题目均为 $O(N)$。
- **空间复杂度**: 
    - 相交(双指针)、反转、环形、合并(拼接版)：$O(1)$。
    - 回文(栈)、合并(new版)：$O(N)$。

### 4. 总结与反思

- **虚拟头节点 (Dummy Node)**：处理链表合并、删除等涉及头节点可能变动的问题时，永远的神。
- **快慢指针 (Fast & Slow)**：
    - 找中点：注意偶数/奇数长度的停止条件。
    - 判圈：`fast` 走两步，`slow` 走一步。
- **断链与重连**：在改变 `cur->next` 之前，永远先用 `temp` 保存 `cur->next`，否则会丢失后续节点（如反转链表）。
