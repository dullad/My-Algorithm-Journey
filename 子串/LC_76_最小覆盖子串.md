### 3. 核心解法：滑动窗口 + 哈希表

**3.1 思路分析**

联想到**滑动窗口**的解法。我们需要用两个哈希表（或者数组）来分别记录目标字符串 `t` 中各字符的出现次数，以及当前窗口中对应字符的出现次数。

整体操作流程分为三步：“**进窗口、判断并收缩窗口、更新答案**”。

1.  **记录目标需求**：
    遍历字符串 `t`，将每个字符及其出现的次数记录在 `t_map` 中。同时设定一个变量 `cnt` 记录当前窗口中已经匹配的**有效字符数量**。

2.  **扩张窗口 (右指针 `r` 向右移)**：
    遍历字符串 `s`。每次将滑动窗口的右边界字符 `s[r]` 加入 `s_map` 中。
    *   **有效计算**：如果加入 `s[r]` 后，`s_map[s[r]] <= t_map[s[r]]`，说明这个字符是我们需要的“有效字符”（还没凑齐目标数量），此时匹配总数 `cnt++`。

3.  **收缩窗口 (左指针 `l` 向右移)**：
    当目前窗口左边界的字符 `s[l]` 是**赘余字符**时，我们就应该尝试缩小窗口。
    *   什么是赘余字符？即它在窗口内的数量已经超过了目标字符串 `t` 中需要的数量（`s_map[s[l]] > t_map[s[l]]`），或者它根本不在 `t` 中对应的数量（`t_map[s[l]] == 0` 的情况也包含在大于号条件里，因为未放入 map 时值为 0）。
    *   遇到赘余字符时，右移左指针 `l++`，并从 `s_map` 中减去对应的数量，直到位于 `l` 的字符是刚好满足 `t` 需求的（不可替代的）。

4.  **记录并更新结果**：
    当 `cnt == t.size()` 时，代表当前窗口已经完全覆盖了 `t` 的字符要求。此时尝试更新最短的字符串结果 `res`。

**3.2 代码实现 (C++)**
```C++
class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map <char, int> s_map, t_map;
        string res;
        // 1. 初始化，统计目标字符串 t 的字符频次
        for(int i = 0; i < t.length(); i++){
            t_map[t[i]]++;
        }
        
        int cnt = 0; // 记录当前窗口内有效匹配的字符总数
        
        for(int r = 0, l = 0; r < s.size(); r++){
            // 2. 右边界字符进入窗口
            s_map[s[r]]++;
            
            // 加入后，如果仍在需要的数量范围内，有效匹配数 + 1
            if(s_map[s[r]] <= t_map[s[r]]) {
                cnt++;
            }
            
            // 3. 收缩窗口：如果左边界字符是“多余”的，就将其移出窗口
            while(s_map[s[l]] > t_map[s[l]]) {
                s_map[s[l++]]--;
            }
            
            // 4. 当有效匹配数达到 t 的长度时，说明找到了一个合法窗口
            if(cnt == t.size()){
                // 若这是第一个匹配到的，或者当前窗口长度更短，则更新 res
                if(res.empty() || r - l + 1 < res.size()){
                    res = s.substr(l, r - l + 1);
                }
            }
        }
        return res;
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(M + N)$。快慢指针都只会从头到尾遍历最多一次，不会回退，因此总时间消耗与长度呈线性。
- **空间复杂度**: 字符集大小，常数级别

### 5. 总结与反思
- **巧妙化解**
    *   `cnt`变量直接记录了有效字符数量，通过 `s_map[s[r]] <= t_map[s[r]]` 可以判断新进入窗口的字符是否是所需要的，避免了每次都要遍历整个哈希表验证两个表是否符合覆盖关系。
    *   收缩条件极其精简：while中的`s_map[s[l]] > t_map[s[l]]`同时处理两种情况：1. 当前字符太多了 2. `t`中压根不存在当前字符。
