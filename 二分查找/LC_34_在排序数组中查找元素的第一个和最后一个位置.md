### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")

*在这里详细记录你最开始的想法，哪怕是错误的。*
先二分找到target的位置，然后从pos开始向左向右分别找到target的左边界和右边界。
易错点： 在找到目标值后，使用 while 或 for 循环向两边线性扩散。虽然逻辑是对的，但在极端情况下（例如整个数组全是 target），线性扩散会让时间复杂度直接退化成 O(N)，从而不满足题目严格要求的 O(log N) 复杂度。

### 3. 核心解法：哈希表

**3.1 思路分析**
破局点： 将寻找“区间”拆解为两次独立的二分查找：一次专门找左边界（即第一个出现的位置），一次专门找右边界（最后一个出现的位置）。

找左边界：当 nums[mid] == target 时，不停止，而是缩小右边界 right = mid - 1，继续向左逼近。

找右边界：当 nums[mid] == target 时，不停止，而是缩小左边界 left = mid + 1，继续向右逼近。

**3.2 代码实现 (C++)**
```c++
class Solution {
public:
    vector<int> searchRange(vector<int>& nums, int target) {
        vector<int> res(2, -1);
        int left = 0, right = nums.size() - 1;
        int pos = -1;
        while(left <= right){
            int mid = (left + right) / 2;
            if(nums[mid] == target){
                pos = mid;
                break;
            }else if(nums[mid] > target){
                right = mid - 1;
            }else{
                left = mid + 1;
            }
        }
        for(int i = pos; i < nums.size(); i++){
            if(nums[i] == target) res[1] = i;
            else break;
        }
        for(int i = pos; i >= 0; i--){
            if(nums[i] == target) res[0] = i;
            else break;
        }
        return res;
    }
};
```
```c++ 优化版代码：严格 O(log N) 的双二分法
class Solution {
public:
    vector<int> searchRange(vector<int>& nums, int target) {
        return {findBound(nums, target, true), findBound(nums, target, false)};
    }

private:
    int findBound(vector<int>& nums, int target, bool isFirst) {
        int left = 0, right = nums.size() - 1;
        int bound = -1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) {
                bound = mid; // 记录当前找到的 target 位置
                if (isFirst) right = mid - 1; // 找左边界，继续向左逼近
                else left = mid + 1;          // 找右边界，继续向右逼近
            } else if (nums[mid] > target) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return bound;
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(log N)$。执行了两次二分查找，每次的时间复杂度均为 $O(log N)$，整体依然是 $O(log N)$（注意：原版线性扩散法最坏为 $O(N)$）。
- **空间复杂度**: $O(1)$。

### 5. 总结与反思

- 找边界问题必须严格执行两次二分查找，切忌在二分查找到目标值后使用线性扩散。
