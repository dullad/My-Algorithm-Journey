### 3. 核心解法：哈希表

**3.1 思路分析**
破局点： 寻找旋转数组中的最小值，无需维护外部最小值变量，直接将 nums[mid] 与最右侧端点 nums[right] 进行比较。因为最小值肯定出现在无序侧的突变右侧。

- 若 nums[mid] < nums[right]：说明 mid 到 right 是严格递增的，最小值一定在 mid 的左侧（甚至可能就是 mid 本身），所以 right = mid。

- 若 nums[mid] > nums[right]：说明左边是有序的，发生旋转的“悬崖”一定在 mid 的右侧，最小值也在右侧，所以 left = mid + 1。

**3.2 代码实现 (C++)**
```c++
class Solution {
public:
    int findMin(vector<int>& nums) {
        int left = 0, right = nums.size() - 1;
        int ans = nums[0];
        while(left <= right){
            int mid = (left + right) / 2;
            if(nums[mid] < ans) ans = nums[mid];
            else{
                if(nums[mid] >= nums[left]){
                    if(nums[left] < ans) ans = nums[left];
                    else{
                        left = mid + 1;
                    }
                }
                else{
                    if(nums[mid + 1] < ans) ans = nums[left];
                    else{
                        right = mid;
                    }
                }
            }
        }
        return ans;
    }
};
```

```c++ 极其精简的右边界比较
class Solution {
public:
    int findMin(vector<int>& nums) {
        int left = 0, right = nums.size() - 1;
        // 注意这里的条件是 left < right，而不是 <=，为了防止 right = mid 时死循环
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] < nums[right]) {
                // 最小值在左半部分，或者是 mid 本身
                right = mid;
            } else {
                // 最小值一定在右半部分（不包含 mid，因为 nums[mid] 已经大于右边界了）
                left = mid + 1;
            }
        }
        // 循环结束时 left == right，指向的就是最小值所在位置
        return nums[left];
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(log(N))$。
- **空间复杂度**: $O(1)$。

### 5. 总结与反思

- 旋转数组求最小值，无脑将中点 mid 与右边界 right 对比，以此来确定“悬崖（最小值）”所在的半区。
