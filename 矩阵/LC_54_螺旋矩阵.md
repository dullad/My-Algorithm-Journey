### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")

*在这里详细记录你最开始的想法，哪怕是错误的。*

*   **边界重叠**：在长方形矩阵中，最后一次循环可能只够走“右”和“下”，如果代码逻辑是强制走完一圈（右下左上），会导致重复访问或者数组越界。


### 3. 核心解法：模拟法（设定四个边界）

**3.1 思路分析**

1.  **定义边界**：我们需要四个变量来限制当前的遍历范围：
    *   `top`：上边界，初始为 `0`
    *   `bottom`：下边界，初始为 `m-1`
    *   `left`：左边界，初始为 `0`
    *   `right`：右边界，初始为 `n-1`
2.  **动作循环**：按照 **“右 $\to$ 下 $\to$ 左 $\to$ 上”** 的顺序不断循环。
3.  **核心技巧（破局思路）**：
    *   **收缩边界**：每遍历完一条边，立刻收缩对应的边界。比如，遍历完最上面一行后，`top` 应该加 1，表示下一圈的上边界下移了。
    *   **立即检查 (Check Immediately)**：这是代码最精髓的地方。在每次收缩边界后，**立刻** 检查当前边界是否越界（例如 `if(++top > bottom) break;`）。如果越界，说明已经遍历完成，不需要再走下面的步骤了。这完美解决了长方形矩阵在遍历结束时只剩一行或一列的情况。


**3.2 代码实现 (C++)**

```c++
class Solution {
public:
    vector<int> spiralOrder(vector<vector<int>>& matrix) {
        int m = matrix.size();
        int n = matrix[0].size();
        vector<int> res(m * n, 0);
        int count = 0, top = 0, bottom = m - 1, left = 0, right = n - 1;
        while(true){
            for(int i = left; i <= right; i++){
                res[count++] = matrix[top][i];
            }
            if(++top > bottom) break; // 收缩上顶

            for(int i = top; i <= bottom; i++){
                res[count++] = matrix[i][right];
            }
            if(--right < left) break; // 收缩右边

            for(int i = right; i >= left; i--){
                res[count++] = matrix[bottom][i];
            }
            if(--bottom < top) break; // 收缩底边

            for(int i = bottom; i >= top; i--){
                res[count++] = matrix[i][left];
            }
            if(++left > right) break; // 收缩左边
        }
        return res;
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(MN)$。
- **空间复杂度**: $O(1)$。

### 5. 总结与反思

- **模拟题的通法**：对于这种“转圈”或者“方向改变”的题目，显式地定义边界变量通常比维护当前坐标 (x, y) 更清晰。
- **Break 的时机**：一定要注意，螺旋矩阵并不是每次都能走完“右下左上”四个方向的。当矩阵只有一行时，走完“右”就该结束了。因此，每走完一个方向就判断一次是否结束。
