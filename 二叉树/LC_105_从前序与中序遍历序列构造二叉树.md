# LeetCode 105. 从前序与中序遍历序列构造二叉树

### 1. 题目链接

[LeetCode 105. 从前序与中序遍历序列构造二叉树](https://leetcode.cn/problems/construct-binary-tree-from-preorder-and-inorder-traversal/description/?envType=study-plan-v2&envId=top-100-liked)

### 2. 我的初次思路与遇到的问题
- 初次尝试直接用 `vector` 切片提取子段传入递归去跑，这种解法极易在参数处理中丢失。且由于频繁在每次递归中开辟、切割、拷贝新的 `vector` 数组，也引入了大量额外的时间和空间开销。
- 快速寻找 `preorder` 首元素在 `inorder` 内的位置是一个瓶颈。若采用每次线性的 O(N) 遍历查找，极端情况下这道题会退化至 $O(N^2)$ 的复杂度。

### 3. 核心解法：分治法 + 传下标闭区间（配合哈希表优化）

**思路分析**
- 构造二叉树的核心在于找准“根节点”，并且切割划分出左右子树的“归属范围”。
- 根据性质：**前序遍历**区间的最左端必定是其所在子树的根节点。
- 拿到该根对应值以后，在**中序遍历**中找到该值的位置：它的左侧就是左子树的所有节点大军，右侧就是右子树的所有节点。
- 利用找出的左树元素数量长度 `l_size` 可以很轻松推算出在接下来的前序中该如何划分布局。
- 用全局哈希表 `unordered_map<int, int>` 记录中序遍历序列中每个值和其对应的索引下标，使得搜寻根节点这步开销从 O(N) 降低到 O(1)。
- 在传参上，完全舍弃重新构建 `vector` 的写法，直接将闭区间 `[pre_l, pre_r]` 和 `[in_l, in_r]` 四个边界标传下去以做到常量空间开销。

**代码实现 (C++)**

```cpp
class Solution {
    unordered_map<int, int> index;
    TreeNode* build(const vector<int>& pre, const vector<int>& in, int pre_l, int pre_r, int in_l, int in_r){
        if(pre_l > pre_r) return nullptr;
        TreeNode* root = new TreeNode(pre[pre_l]); // 注意以此层前序起点为根
        int root_pos = index[pre[pre_l]]; // 哈希查找其中序所在位置
        int l_size = root_pos - in_l; // 得出左子树含有的节点数量
        
        // 带着边界继续往下建设
        root->left = build(pre, in, pre_l + 1, pre_l + l_size, in_l, root_pos - 1);
        root->right = build(pre, in, pre_l + 1 + l_size, pre_r, root_pos + 1, in_r);
        return root;
    }
public:
    TreeNode* buildTree(vector<int>& preorder, vector<int>& inorder) {
        int n = preorder.size();
        for(int i = 0; i < n; i++){
            index[inorder[i]] = i;
        }
        return build(preorder, inorder, 0, n-1, 0, n-1);
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(N)$。构建初始用哈希表记录中序值需 $O(N)$；递归时每个点构建恰好均执行一次且耗时为 $O(1)$ 查找。总体 $O(N)$。
- **空间复杂度**: $O(N)$。哈希表占用 $O(N)$ 级别辅助内存，此外对应回溯递归树高的系统调用栈。

### 5. 总结与反思

- 重建二叉树问题，找准各大序列特点“根在哪里”、“左树块大小”，“闭区间的运算” 是万能的拆分公式。
- “利用维护下标作为界定游标区间加上辅助哈希表”是降低本类考题空间开销与重复查找惩罚的最佳优化配置。
