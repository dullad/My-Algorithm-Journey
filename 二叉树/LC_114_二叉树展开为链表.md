### 3. 核心解法：分治法与后序处理

**思路分析**
- 递归函数 `flat_node(root)` 展开以 `root` 为根的树并返回**展开后的链表尾部节点**。
- 分治思想：先把左子树和右子树分别递归展开。拿到左子树展开后的尾节点 `leftTail` 和右子树展开后的尾节点 `rightTail`。
- 接线拼接：如果存在左子树，将左子树整体插入到 `root` 和 `root->right` 之间。最后记得将 `root->left` 置空。
- 返回值规划：优先返回右子树的尾节点作为总链表尾端，如果没有右子树则返回左子树的尾端，若是叶子节点则返回其自身。

**代码实现 (C++)**

```cpp
class Solution {
    TreeNode* flat_node(TreeNode* root){
        // 明确返回值是什么，就是当前节点展开后的链表尾部节点
        if(!root || (!root->left && !root->right)) return root;
        TreeNode* leftTail = nullptr;
        TreeNode* rightTail = nullptr;
        if(root->left) leftTail = flat_node(root->left);
        if(root->right) rightTail = flat_node(root->right);
        if(leftTail){
            leftTail->right = root->right;
            root->right = root->left;
            root->left = nullptr; // 别忘了左侧要清空
        }
        if(rightTail) return rightTail;
        else return leftTail;
    }
public:
    void flatten(TreeNode* root) {
        flat_node(root);
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(N)$，每个节点被访问一次，常数次指针调整操作。
- **空间复杂度**: $O(H)$，即递归栈的深度，最坏情况退化为一层层的单链时为 $O(N)$，平均情况为 $O(\log N)$。符合进阶中不额外再建立 `vector` 的原地操作潜规则。

### 5. 总结与反思

- 遇到需要修改左右子树结构并串联的问题，**后序遍历（分治合并思想）**是很好的切入点，先底解决并拿上返回值，再在根节点做最后的拼接。
- 需要给递归函数明确清晰的“返回值定义”（这一题是返回当前延展后的尾部节点）。
