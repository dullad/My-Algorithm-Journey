# LeetCode 543. 二叉树的直径

### 1. 题目链接

[LeetCode 543. 二叉树的直径](https://leetcode.cn/problems/diameter-of-binary-tree/description/?envType=study-plan-v2&envId=top-100-liked)

### 2. 我的初次思路与遇到的问题
- 初次尝试直接按“当前直径是否取左右节点最大直径，还是横跨当前节点”进行暴力递归：max(max(dL, dR), depL + depR)。这种解法为了计算深度又开了额外的高频 maxDepth 遍历叠加，使得总体时间复杂度退化到了 (N^2)$。

### 3. 核心解法：后序遍历计算深度的夹带私货优化

**思路分析**
- 我们注意到：任一节点的最长左单链跟右单链结合就能构成一条经过它的穿线“直径”（线段数为 左深度 + 右深度）。
- 全局树的最长直径，就是所有的节点上产生的这段直径中挑出的最大值。
- 通过设计个**后序遍历**的搜底层函数去向上返回深度供层级用，同时在内部加入一句 ns = max(ans, L + R)；这种将求直径和返回深度统一放到一次遍历之中的做法能成功完成 (N^2)$ 降级至 (N)$。

**代码实现 (C++)**

```cpp
class Solution {
    int ans = 0;
    int maxDepth(TreeNode* root){
        if(root == nullptr) return 0;
        int L = maxDepth(root->left);
        int R = maxDepth(root->right);
        ans = max(ans, L + R); // 在求深度的过程中，夹杂统计其经过这点的直径做对比
        return max(L, R) + 1;  // 向上层正常交代返回自身这枝的最大节点深度
    }
public:
    int diameterOfBinaryTree(TreeNode* root) {
        maxDepth(root);
        return ans;
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: (N)$。整树仅进行一次单向无回退DFS巡回。
- **空间复杂度**: (H)$。对应回溯压栈深度，最好为(\log N)$（平衡树），最差(N)$。

### 5. 总结与反思

- 将求解的独立记录（最大值）提取成全局或者是类内引用传参维护是一个很巧妙的避开二次查找的常见优化。此题和二叉树求最大路径和（LeetCode 124）思路一脉相承。
