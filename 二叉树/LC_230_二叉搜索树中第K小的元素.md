# LeetCode 230. 二叉搜索树中第 K 小的元素

### 3.1 核心解法1：中序遍历

**思路分析**
利用 BST 的性质，其中序遍历恰好是升序序列。因此只要进行普通的中序遍历，并用一个全局或类成员变量作为计数器 `cnt`，记录当前访问到的是第几个节点。当 `cnt == k` 时保存当前节点值即为答案。

```cpp
class Solution {
    int cnt = 0;
    int res;
    void inorder(TreeNode* root, int k){
        if(!root) return;
        inorder(root->left, k);
        cnt++;
        if(cnt == k) res = root->val;
        inorder(root->right, k);
    }
    
public:
    int kthSmallest(TreeNode* root, int k) {
        inorder(root, k);
        return res;
    }
};
```

### 3.2 核心解法2：迭代遍历

**思路分析**
利用栈模拟中序遍历的过程。一路向左将左节点入栈，当左边走到底时，弹出栈顶元素（即为当前最小的元素），计数器加一。如果此时计数达到 `k`，直接返回该节点值，提前终止遍历；否则转向其右子树继续流程。

**代码实现 (C++)**

```cpp
class Solution {
public:
    int kthSmallest(TreeNode* root, int k) {
        stack<TreeNode*> st;
        TreeNode* cur = root;
        int cnt = 0;
        while(cur || !st.empty()){
            while(cur){
                st.push(cur);
                cur = cur->left;
            }
            cur = st.top();
            st.pop();
            cnt++;
            if(cnt == k) return cur->val;
            cur = cur->right;
        }
        return -1;
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: `$O(H + k)$`。其中 `H` 是树的高度，找到最小元素需要 $O(H)$，随后再中序遍历找 $k-1$ 个元素需要 $O(k)$。
- **空间复杂度**: `$O(H)$`。对应回溯或迭代压栈深度。

### 5. 总结与反思

- BST 第 K 小或大元素首选中序遍历。递归写法实现简单，但很多时候无法很方便地中断提前返回。此题使用迭代写法可以随时在找到结果后提前退出（短路操作），在大型 BST 中求极小 K 的效率会明显提升。
