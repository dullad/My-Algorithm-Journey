# LeetCode 236. 二叉树的最近公共祖先

### 1. 题目链接

[LeetCode 236. 二叉树的最近公共祖先](https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/description/?envType=study-plan-v2&envId=top-100-liked)

### 2. 我的初次思路与遇到的问题
- 初次设计思路较为混乱，原本想去递归寻找 p 和 q 分别是否在左子树或右子树里，如果是则不断交替推进判断，但逻辑分支极难理清收尾而且很多情况都无从考虑。想要在一个普通方法里去实现是极其繁杂且死板的。

### 3. 核心解法：后序遍历接收并上抛目标

**思路分析**
- 将该函数的实质意图理解成：在此子树下寻找 p 和 q 并且将找到的节点当作结果一路上抛上去。
- **阶段终止条件**：遇到了 `nullptr`（落空返回）、遇到了 `p` 或 `q`。
- **通过后序判断左右两侧反馈**：
  1. 向左下方派发去左子树搜寻得到 `leftCommon` 收获反馈；向右探索得到 `rightCommon` 收获反馈。
  2. 如果他发现两侧**各抛出了一个目标值/祖先反馈值**（即两者均不为 `nullptr`），那就立刻证明 `p` 和 `q` 一个在它的左边、一个在它的右边。所以它本身节点就是这个最近公共祖先，那就将自身 `root` 作为一个更高大更尊贵的线索传向上方！
  3. 如果只在一侧发现了，就将找到的那侧的搜寻结果**直截了当传接给更上部分**

**代码实现 (C++)**

```cpp
class Solution {
public:
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        // 如果碰头目标元素（或者落到底层）了直接返回
        if(!root || root == p || root == q) return root;
        
        TreeNode* leftCommon = lowestCommonAncestor(root->left, p, q);
        TreeNode* rightCommon = lowestCommonAncestor(root->right, p, q);
        
        // 只有一侧传来结果，说明另一个可能在它的下属（或者单纯在这个面下面没有），原样代传。
        if(!leftCommon) return rightCommon;
        if(!rightCommon) return leftCommon;
        
        // 两侧都有传递响应反馈证明他们一左一右，自身就是这个交汇点的最近祖先节点。
        return root;
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(N)$。最坏情况下会走完全局一次遍历去寻觅到末尾。
- **空间复杂度**: $O(H)$。递归回溯压栈限制对应当前二叉树本身的树度。

### 5. 总结与反思

- 此题灵魂是在于将函数自身递归的“返回值”附着非常高级灵活的功能性质：它不仅会传找到的首个单节点，还会在一旦促成两侧夹击之时直接自身变为公共祖先升级上传。这是自底向上极其顺滑的机制理念设计。
