### 3. 核心解法

**3.1 思路分析**

这道题的难点在于时间复杂度`O(N)`且空间复杂度`O(1)`。无法使用哈希表，且必须利用数组本身作为哈希表。
希望数组变成一个有序哈希表，数值`x`放在对应下标`x-1`的位置上。
算法流程：
1. 归位（置换）：遍历数组，只要`nums[i]`在`[1, N]`范围内，且没有放在正确的位置上，我们就把它交换到应该去的地方，即下标`nums[i]-1`。
   - 特别注意：如果目标位置上已经有了正确的数，就不需要交换，避免死循环。
2. 查找：再次遍历数组，第一个满足nums[i] != i+1的位置，说明缺失了正数i+1。
3. 如果所有位置都对的上，说明数组是`[1, 2, ..., N]`，缺失的是`N+1`。

**3.2 代码实现 (C++)**

```c++
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        for(int i = 0; i < nums.size(); i++){
            while(nums[i] != i + 1){
                if(nums[i] <= 0 || nums[i] > nums.size() || nums[i] == nums[nums[i] - 1]) break;
                int to = nums[i] - 1;
                nums[i] = nums[to];
                nums[to] = to + 1;
                // swap(nums[i], nums[nums[i] - 1]);
            }
        }
        for(int i = 0; i < nums.size(); i++){
            if(nums[i] != i + 1){
                return i + 1;
            }
        }
        return nums.size() + 1;
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(N)$。
- **空间复杂度**: $O(1)$。

### 5. 总结与反思

- **萝卜坑思想（原地哈希）**：当题目要求 O(1) 空间，且涉及数字范围和索引有关系时，优先考虑将数组元素交换到对应下标处。
- **易错点**：
    *   **死循环**：交换时必须判断 `target` 位置的数是否已经正确，如果 `nums[i] == nums[nums[i]-1]`，说明有重复元素，必须停止交换，否则会无限 `swap`。。
    *   界条件：注意判断正数范围 `nums[i] > 0` 和 `nums[i] <= n`，因为超过 n 的数我们不关心（它们注定缺失在 `[1, n]` 范围内或不影响 `n+1` 的判断）
