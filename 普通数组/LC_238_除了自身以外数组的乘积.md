### 3. 核心解法：前缀积与后缀积

**3.1 思路分析**

既然不能用除法，我们不妨换个角度思考：索引 `i` 处的结果，实际上等于它**左边所有元素的乘积**乘以它**右边所有元素的乘积**。

1.  **朴素做法 ($O(N)$ 空间)**：
    我们可以开辟两个数组 `L` 和 `R`。
    *   `L[i]` 表示索引 `i` 左侧所有元素的乘积。
    *   `R[i]` 表示索引 `i` 右侧所有元素的乘积。
    通过两次遍历分别填充出 `L` 和 `R`，最后对于任意位置 `i`，`ans[i] = L[i] * R[i]`。

2.  **空间优化技巧 ($O(1)$ 空间)**：
    题目中提到，作为返回结果的数组不算在额外空间复杂度中。所以我们可以直接用结果数组 `ans` 来暂存数据，从而省去 `L` 和 `R` 数组！
    *   **第一步（求左侧积）**：先用 `ans` 数组充当 `L` 数组的功能，从左到右遍历，用一个变量 `pre` 记录当前位置之前的累乘积，赋值给 `ans[i]`。
    *   **第二步（融合右侧积）**：从右向左遍历数组，用一个变量 `suf` 记录当前位置之后的累乘积。在遍历过程中，将之前存好的“左侧积” `ans[i]` 直接乘上滚动更新的 `suf`，即可原地算出最终答案。


**3.2 代码实现 (C++)**

```c++
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        vector<int> ans(nums.size(), 1);
        // vector<int> a(nums.size(), 1);
        // vector<int> b(nums.size(), 1);
        // for(int i = 0; i < nums.size() - 1; i++){
        //     a[i + 1] = a[i] * nums[i];
        // }
        // for(int j = nums.size() - 1; j >= 1; j--){
        //     b[j - 1] = b[j] * nums[j];
        // }
        // for(int i = 0; i < nums.size(); i++){
        //     ans[i] = a[i] * b[i];
        // }

        // 空间复杂度为O(1)的方法：直接用ans先作为前缀表
        int pre = 1, suf = 1;
        for(int i = 0; i < nums.size(); i++){
            ans[i] = pre;
            pre *= nums[i];
        }
        for(int j = nums.size() - 1; j >= 0; j--){
            ans[j] *= suf;
            suf *= nums[j];
        }
        return ans;
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(N)$。
- **空间复杂度**: $O(1)$。

### 5. 总结与反思

- **前缀/后缀思想**：当题目需要计算“除了某元素自身以外的所有元素之XX”或者处理区间求和/求积的问题时，前缀和 / 前缀积 是非常经典且高效的方法。它把 O(N) 的每次查询优化到了 O(1) 。
- **空间优化的精髓（复用变量）**：当发现多次遍历的辅助数组最终只为了合并结果时，可以尝试将其中的某些计算改用“滚动变量动态维护”，并将结果就地 (In-place) 覆盖在输出数组上，以达到降低空间复杂度的目的。
