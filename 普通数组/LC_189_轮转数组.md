### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")

*在这里详细记录你最开始的想法，哪怕是错误的。*

1.  **队列辅助法**：先维护一个队列，倒叙将数组入队。每一次轮转就是一次出队，然后将出队元素入队。轮转完后用队列去更新数组。
    *   *问题*：空间复杂度偏高，且进出队列操作增加了不必要的开销，虽然能过，但不够优雅。
2.  **暴力原地轮转（每次移1位，移k次）**：维护 `temp1` 和 `temp2`，每一次轮转遍历，将当前位置数保存在 `temp2` 中，然后用 `temp1` 更新当前位置，接着用 `temp2` 更新 `temp1`。循环一次则轮转一次，`k` 为几就外层循环几次即可。
    *   *问题*：逻辑完全正确，不需要额外空间。但**时间复杂度太高，会导致超时 (TLE)**，为 $O(k \times N)$。

### 3. 核心解法：数组翻转法 (最优解)

**3.1 思路分析**

如果不使用额外空间，又要在 $O(N)$ 时间复杂度内完成轮转，最经典的技巧是**整体/局部翻转法**。

我们可以把原数组分为两部分：
- 前半部分：不需要跨越数组尾部的部分（长度为 $n - k$）
- 后半部分：需要移动到数组头部的部分（长度为 $k$）

以 `nums = [1,2,3,4,5,6,7], k = 3` 为例：
我们其实就是想把后半部分 `[5,6,7]` 挪到前面，变成 `[5,6,7, 1,2,3,4]`。

操作步骤如下（只需要三次翻转）：
1.  **反转所有元素**：`[7,6,5,4,3,2,1]` (此时后半部分到了前面，但内部顺序是反的)
2.  **反转前 $k$ 个元素**：`[5,6,7, 4,3,2,1]` (前 $k$ 个元素的顺序被矫正了)
3.  **反转剩余的 $n - k$ 个元素**：`[5,6,7, 1,2,3,4]` (后面部分的顺序也被矫正了，得到最终答案)

*注意：当 $k$ 大于数组长度 $n$ 时，轮转 $n$ 次等于没有轮转。所以实际有意义的轮转步数是 $k \pmod n$。*

**3.2 代码实现 (C++)**

```c++
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        queue<int> num_que;
        for(int i = nums.size() - 1; i >= 0; i--){
            num_que.push(nums[i]);
        }
        
        while(k > 0){
            int temp = num_que.front();
            num_que.pop();
            num_que.push(temp);
            k--;
        }
        for(int i = nums.size() - 1; i >= 0 ;i--){
            nums[i] = num_que.front();
            num_que.pop();
        }
    }
};
```

```c++
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int n = nums.size();
        k = k % n; // 防止 k 大于数组长度
        if (k == 0) return; // 刚好轮转整数倍，不用变
        
        // 1. 翻转整个数组
        reverse(nums.begin(), nums.end());
        
        // 2. 翻转前 k 个元素
        reverse(nums.begin(), nums.begin() + k);
        
        // 3. 翻转后面剩余的元素
        reverse(nums.begin() + k, nums.end());
    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(N)$。
- **空间复杂度**: $O(1)$。

### 5. 总结与反思

- **取模的思想**：遇到“循环”、“轮转”、“周期”等字眼，第一下意识反应应该是用取余操作 % 来消除无效的循环圈数。
- **数组翻转**：对于所有**“交换数组前、后两块区间”的问题，都可以尝试思考三次翻转的技巧。这个思维在字符串循环平移等题目中也广泛适用。C++ 的 `std::reverse(begin, end) `也是必须要掌握的高频 API（注意是左闭右开区间）。
