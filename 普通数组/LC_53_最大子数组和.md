用 `preMax` 记录在当前数字前最大子数组之和，当当前数字引入后，将一直到当前数字的最大子数组之和更新到`preMax`中，将`preMax`与`maxAns`比较更新`maxAns`，得到全局的最大子数组之和。

### 3. 核心解法：双指针

**3.1 思路分析**

这是一道非常经典的动态规划题目，最优解法通常被称为 **Kadane 算法**。

1.  **状态定义**：
    假设 `dp[i]` 表示以 `nums[i]` 结尾的连续子数组的最大和。注意，这个子数组**必须**包含 `nums[i]`，这样才能保证后续状态可以与之相连凑成“连续”。

2.  **状态转移方程**：
    对于当前的元素 `nums[i]`，我们只有两种选择：
    *   **加入前一个子数组**：如果之前的连续子数组和 `dp[i-1]` 为正数，那么加上它是有收益的。此时 `dp[i] = dp[i-1] + nums[i]`。
    *   **重新开始**：如果之前的连续子数组和 `dp[i-1]` 是负数（或者零），那么带着它只会拖累当前的数字。不如从 `nums[i]` 开始重新计算一个新子数组。此时 `dp[i] = nums[i]`。
    
    综合起来，状态转移方程就是：`dp[i] = max(dp[i-1] + nums[i], nums[i])`。

3.  **空间优化**：
    计算 `dp[i]` 时只需要用到前一个状态 `dp[i-1]`。因此，我们不需要开辟一个完整的 `dp` 数组，而是可以用一个单一的变量 `preMax` 来不断滚动记录“以当前元素结尾的最大子数组和”即可。这就是空间优化版的实现。同时，再维护一个全局的 `maxAns` 来记录所有的 `preMax` 里出现过的最大值。

**3.2 代码实现 (C++)**

```c++
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int preMax = 0;
        int maxAns = nums[0];
        for(const auto &x: nums){
            preMax = max(preMax + x, x);
            maxAns = max(maxAns, preMax);
        }
        return maxAns;

    }
};
```

### 4. 复杂度分析

- **时间复杂度**: $O(N)$。
- **空间复杂度**: $O(1)$。

### 5. 总结与反思

- **滚动数组思想降维**：当我们发现动态规划中，第 i 个状态只依赖于第 i - 1 个状态，可以用少量变量滚动记录。
- **初始值的设置**：全局最大值 maxAns 一定要初始化为数组的第一个元素 nums[0]。千万不能初始化为 0，否则当整个数组都是负数时（比如 [-2, -1]），会错误地返回 0。
