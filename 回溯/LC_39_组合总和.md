# LeetCode 39. 组合总和

### 1. 题目链接
[39. 组合总和](https://leetcode.cn/problems/combination-sum/description/?envType=study-plan-v2&envId=top-100-liked)

### 2. 我的初次思路与遇到的问题
原代码在每一层递归都用 `for` 循环重复求 `temp_sum`，效率极低。更好的做法是将当前总和直接作为一个参数传下去（或者利用 target 减小传下去）。

### 3. 核心解法：带参数优化的回溯

**3.1 思路分析**
同一元素可无限制重复选取。
- 向下递归时，下一层的起始索引**不是 `i+1` 而是 `i`（自身）**。
- 利用倒扣法做减法控制和，如果 `target < 0` 就剪枝返回，如果 `target == 0` 就是找到了和的答案。

**3.2 代码实现 (C++)**
```cpp
class Solution {
    vector<vector<int>> res;
    vector<int> comb;
    void backtrace(vector<int>& candidates, int start, int remain){
        if(remain < 0) return; // 剪枝优化，超了
        if(remain == 0){
            res.push_back(comb);
            return;
        }
        for(int i = start; i < candidates.size(); i++){
            comb.push_back(candidates[i]);
            // 下一层仍从 i 开始，因为元素可被重复选中
            backtrace(candidates, i, remain - candidates[i]);
            comb.pop_back();
        }
    }
public:
    vector<vector<int>> combinationSum(vector<int>& candidates, int target) {
        backtrace(candidates, 0, target);
        return res;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: `$O(S)$`，S 为所有可行解的长度之和。最坏指数级 `$O(2^{target/min\_val})$`。
- **空间复杂度**: `$O(target / min\_val)$`。最差完全取极小值使得递归层数最大。

### 5. 总结与反思
- **组合可重复模板**：向下的索引传入的是 `i`，而不是 `i+1`（区别普通限制）。
- **求和回溯的常规优化**：向下传入减去的差值做退出条件（`remain`），远比使用循环临时求和轻且快。
