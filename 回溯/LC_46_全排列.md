# LeetCode 46. 全排列

### 1. 题目链接
[46. 全排列](https://leetcode.cn/problems/permutations/description/?envType=study-plan-v2&envId=top-100-liked)

### 2. 我的初次思路与遇到的问题
使用辅助数组 `used` 记录当前元素是否在排列中已被使用。

### 3. 核心解法：无重复回溯

**3.1 思路分析**
排列问题与组合问题的不同之处在于：排列有顺序，每次选择都必须从头遍历数组（`i = 0`），而不是从 `start` 开始。通过记录数组 `flag_used` 来保证同一个数字在同一次深搜中不被重复选择。

**3.2 代码实现 (C++)**
```cpp
class Solution {
    vector<vector<int>> res;
    vector<int> path;
    void traceback(vector<int>& nums, vector<bool>& flag_used){
        if(path.size() == nums.size()){
            res.push_back(path);
            return;
        }
        for(int i = 0; i < nums.size(); i++){
            if(flag_used[i]) continue; // 用过则跳过
            path.push_back(nums[i]);
            flag_used[i] = true;
            traceback(nums, flag_used);
            path.pop_back(); 
            flag_used[i] = false;
        }
    }
public:
    vector<vector<int>> permute(vector<int>& nums) {
        vector<bool> flag_used(nums.size(), false);
        traceback(nums, flag_used);
        return res;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: `$O(N \times N!)$`。共 `N!` 个排列，每个排列的生成或保存需要 `$O(N)$`。
- **空间复杂度**: `$O(N)$`。递归栈深度为N，额外的 `used` 数组和 `path` 也均为 `$O(N)$`。

### 5. 总结与反思
- 排列问题：**每次都从 0 遍历**，靠 `used` 数组去重。
- 组合/子集问题：**每次从 startIndex 遍历**，天然去重，不需要 `used`。
