# LeetCode 79. 单词搜索

### 1. 题目链接
[79. 单词搜索](https://leetcode.cn/problems/word-search/description/?envType=study-plan-v2&envId=top-100-liked)

### 2. 我的初次思路与优化分析
单纯暴搜回溯法。第一版记录用了 `visited` 二维数组去防重，时间和空间消耗变大；优化版本巧妙通过将原字符改成占位符做**自标记**。

### 3. 核心解法：带复用的矩阵 DFS

**3.1 思路分析**
- 利用两层 for 选定网格的所有落脚点当作起点尝试。
- DFS如果匹配到了 `word[index]`，为了防止周围节点扩展向回溯，将自身暂时替换为 `'0'`（非法字符占位），发散向周围四方搜寻 `index+1`。如果死活走不通则最后改回还原（状态重置）。

**3.2 代码实现 (C++)**
```cpp
class Solution {
    int dirs[4][2] = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};
    bool backtrack(vector<vector<char>>& board, int i, int j, const string& word, int index){
        // 从(i,j)出发，能否找到index开始的后续word(包含index位置)
        if(board[i][j] != word[index]) return false;
        if(index == word.length() - 1) return true;
        
        char temp = board[i][j];
        board[i][j] = '0'; // 沉浸式标记：防止下一步回头搜回来
        bool res = false;
        for(auto& dir: dirs){
            int new_i = i + dir[0], new_j = j + dir[1];
            if(new_i >= 0 && new_i < board.size() && new_j >= 0 && new_j < board[0].size()){
                if(backtrack(board, new_i, new_j, word, index + 1)){
                    res = true;
                    break;
                }
            }
        }
        board[i][j] = temp; // 现场还原重置
        return res;
    }
public:
    bool exist(vector<vector<char>>& board, string word) {
        if (board.empty() || board[0].empty()) return false;
        int h = board.size(), w = board[0].size();
        for(int i = 0; i < h; i++){
            for(int j = 0; j < w; j++){
                if(backtrack(board, i, j, word, 0)){
                    return true;
                }
            }
        }
        return false;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: `$O(M \times N \times 3^L)$`。`M,N`为方格长宽。每次走除了回头共有不超过3个方向散开，L为word的长。
- **空间复杂度**: `$O(L)$`。没用辅数组，仅由于递归占用，深度最多延伸至目标单词的长度。

### 5. 总结与反思
- DFS搜索原矩盘的精髓优化：**`记初值 -> 原地防重标记 -> 搜四面八方 -> 还复初值`**。
- `const string&` 的传入非常必要，避免重复的超长深拷贝导致超时。
