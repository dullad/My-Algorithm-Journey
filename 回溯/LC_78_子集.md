# LeetCode 78. 子集

### 1. 题目链接
[78. 子集](https://leetcode.cn/problems/subsets/description/?envType=study-plan-v2&envId=top-100-liked)

### 2. 我的初次思路与遇到的问题
一开始把 `res.push_back(path);` 放到终止条件中，导致只收录到了叶子节点（长度最长的组合）。子集问题的特殊性在于**每一个遍历树上的节点都是合法答案**。

### 3. 核心解法：回溯收集所有节点

**3.1 思路分析**
任何时候（包括进入函数第一时间），当前的 `path` 都必定是一个合法的子集，因此不需要任何条件判断就直接加进 `res`。控制变量 `start` 确保不会取之前选过的元素，避免出现倒序重复子集。

**3.2 代码实现 (C++)**
```cpp
class Solution {
    vector<vector<int>> res;
    vector<int> path;
    void traceback(vector<int>& nums, int start){
        if(start > nums.size()) return; 
        for(int i = start; i < nums.size(); i++){
            path.push_back(nums[i]);
            res.push_back(path); // 每添加一个节点都是一个新子集
            traceback(nums, i + 1);
            path.pop_back();
        }
    }
    
public:
    vector<vector<int>> subsets(vector<int>& nums) {
        res.push_back(path); // 空集也是有效子集
        traceback(nums, 0);
        return res;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: `$O(N \times 2^N)$`。包含 N 个元素的集合共有 `$2^N$` 个子集，每次构造需要 `$O(N)$`。
- **空间复杂度**: `$O(N)$`。递归栈及 `path` 占用空间为 `$O(N)$`。

### 5. 总结与反思
- 子集问题的精髓：**“没到终止条件就开始收获答案”**。每次沿途路过就是一种符合题意的可能。
