# LeetCode 131. 分割回文串

### 1. 题目链接
[131. 分割回文串](https://leetcode.cn/problems/palindrome-partitioning/description/?envType=study-plan-v2&envId=top-100-liked)

### 2. 核心解法：将切割转化为回溯组合

**3.1 思路分析**
切割问题可以类比成找组合的点。每次作决策：“以 `index` 开头，要切到哪个点 `i` 才算是回文？如果这刀切下去是回文，剩下的才允许交给递归继续切。”
通过在 `[index, i]` 区间作回文判定：
- 若为真，提取该段放入结果，进入 `i+1` 递归底层；
- 若为假，继续向右延伸 `i`，找下一个合法的切点。

**3.2 代码实现 (C++)**
```cpp
class Solution {
    vector<vector<string>> res;
    vector<string> part;
    
    // 双指针在原串上直接判断闭区间[left, right]
    bool check(string& s, int left, int right){
        while(left < right){
            if(s[left++] != s[right--]) return false;
        }
        return true;
    }
    
    void backtrack(string& s, int index){
        // 从index开始分割字符串，搜集所有合法回文分割方案
        if(index == s.length()){
            res.push_back(part);
            return;
        }
        for(int i = index; i < s.length(); i++){
            // 如果[index,i]之间的字符构成回文，则加入part，然后从i+1再找。
            if(check(s, index, i)){
                part.push_back(s.substr(index, i - index + 1));
                backtrack(s, i + 1);
                part.pop_back(); // 回退找另一个更长可行解
            }
        }
    }
public:
    vector<vector<string>> partition(string s) {
        backtrack(s, 0);
        return res;
    }
};
```
*(注:代码里的 `check` 做了指针入参改造，省去了大量截取临时字符串反复对冲性能的问题)*

### 4. 复杂度分析
- **时间复杂度**: `$O(N \times 2^N)$`，N为串长，若是如"aaaa"这样的串有巨多的组合方式切，判断以及数组深贴进需耗时。
- **空间复杂度**: `$O(N)$`，用于存储路径栈以及栈帧递归。

### 5. 总结与反思
- 解决切割问题要明白：切割点也是组合选项，其推进形式和标准的组合一模一样。底层只需负责将剩下 `i+1` 继续分割搜罗。
