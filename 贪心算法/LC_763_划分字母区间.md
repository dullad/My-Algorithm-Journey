# LeetCode 763. 划分字母区间

### 1. 题目链接
[https://leetcode.cn/problems/partition-labels/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
max_pos(26, -1)，记录所有字母的最大下标。维护一个pre_end记录上一次划分的末尾，cur_end在遍历过程中实时更新。一旦i=cur_end，则产生一个新的划分，将该长度加入结果，并更新pre_end。
**易错点：** 确定分割条件的条件易错。必须保证当前切片内包含的所有字母，均不会在切片外的后缀字符串中再次出现。

### 3. 核心解法：贪心（哈希最远距）
**3.1 思路分析**
**破局点：** 首先遍历一次字符串，使用哈希表或数组记录每个字符在字符串中出现的最后位置。在第二次遍历分割时，维护当前区间内所有字符的最后出现位置的最大值。当遍历下标等于该最大值时，说明当前区间内的字符已全部完整包含在内，可以进行分割。

**3.2 代码实现 (C++)**
```c++
// 完整写出
class Solution {
public:
    vector<int> partitionLabels(string s) {
        vector<int> max_pos(26, -1);
        vector<int> ans;
        for(int i = 0; i < s.length(); i++){
            max_pos[s[i] - 'a'] = i;
        }
        int pre_end = 0;
        int cur_end = max_pos[s[0] - 'a'];
        for(int i = 0; i < s.length(); i++){
            cur_end = max(cur_end, max_pos[s[i] - 'a']);// 必须先更新cur_end
            if(i == cur_end){
                ans.push_back(1 + cur_end - pre_end);
                pre_end = cur_end + 1;
            }
        }
        return ans;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N)$。需要遍历字符串两次，一次记录最远位置，一次进行区间分割。
- **空间复杂度**: $O(|\Sigma|)$。只使用了大小为26的辅助数组，常数级别空间。

### 5. 总结与反思
利用哈希表或数组预设并记录字符的全局最远边界，在遍历中动态判定当前子串是否满足独立条件。
