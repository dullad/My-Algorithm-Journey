# LeetCode 55. 跳跃游戏

### 1. 题目链接
[https://leetcode.cn/problems/jump-game/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
维护一个当前能达到的最大下标`max_location`，然后遍历数组，用当前位置能跳到的最远下标更新`max_location`。如果遍历到`max_location`以外的位置，说明无法达到。如果`max_location`某次大于数组最大下标，说明可以到达最后一个下标。
**易错点：** 在遍历更新最远到达边界时，如果缺乏 `i > max_location` 的判断，会导致将无法到达的下标也作为起跳点。

### 3. 核心解法：贪心算法
**3.1 思路分析**
**破局点：** 将问题转化为计算可覆盖的最大边界。维护当前能够到达的最远下标，并在遍历每一个可达位置时更新该边界。如果当前遍历的下标超出了已知的最远边界，则说明无法继续前进。

**3.2 代码实现 (C++)**
```c++
class Solution {
public:
    bool canJump(vector<int>& nums) {
        int max_location = 0;
        for(int i = 0; i < nums.size(); i++){
            if(i > max_location) return false;// 关键，首先要判断是否能到达该位置
            max_location = max(max_location, i + nums[i]);
            if(max_location >= nums.size() - 1) return true;
        }
        return true;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N)$。需要遍历数组一次。
- **空间复杂度**: $O(1)$。仅使用了常数个变量。

### 5. 总结与反思
贪心算法的应用体现在每次都尝试更新全局的最大可达范围，而不是关注具体的跳跃路径。
