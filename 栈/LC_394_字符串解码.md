# LeetCode 394. 字符串解码

### 1. 题目链接
[https://leetcode.cn/problems/decode-string/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
顺手把遇到右括号之前的所有字符塞入栈中。遇到右括号时退栈合并得到字符串及数字倍数，反手重新放入栈。
**易错点：** 对重复次数循环解析时数字权值容易反位计算（多位数情况）；而在将解压后子串塞回栈里时，直接 push 会破坏相对顺序。

### 3. 核心解法：单辅助栈处理
**3.1 思路分析**
**破局点：** 将 `[` 视为内层边界进行天然阻断。顺势入栈所有字符，每碰到一次 `]` 则退栈到最近一个 `[`。依次组合剥离出的局部基串与局部频次，将其翻倍组合后**再次以相反顺序塞入栈**，最终得到纯字符串。

**3.2 代码实现 (C++)**
```C++
class Solution {
public:
    string decodeString(string s) {
        string res;
        stack<char> st;
        for(int i = 0; i < s.length(); i++){
            if(s[i] != ']'){// 除了右括号都入栈
                st.push(s[i]);
            }else{// 如果是右括号，需要开始处理，一直处理到左括号
                string temp;
                // 栈中字符取到temp中
                while(!st.empty() && st.top() != '['){
                    temp.push_back(st.top());
                    st.pop();
                }
                if(st.top() == '['){
                    st.pop();
                    // 取出整数，注意会大于9
                    int base = 1, re = 0;
                    while(!st.empty() && st.top() >= '0' && st.top() <= '9'){
                        re += (st.top() - '0') * base;
                        st.pop();
                        base *= 10;
                    }
                    // 复制re遍 入栈
                    for(int j = 0; j < re; j++){
                        for(int k = temp.size() - 1; k >= 0; k--){// 注意出栈入栈顺序
                            st.push(temp[k]);
                        }
                    }
                }
            } 
        }
        while(!st.empty()){
            res.push_back(st.top());
            st.pop();
        }
        reverse(res.begin(), res.end());
        return res;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N)$。主要与最后解码得到的总字符串展开长度有关挂钩。
- **空间复杂度**: $O(N)$。临时使用栈的深度辅助。

### 5. 总结与反思
对于这种涉及嵌套循环、展开结构、且依赖内部就近匹配的模型，入栈出栈+再入栈是非常粗暴但有效的套路。
