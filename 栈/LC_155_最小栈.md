# LeetCode 155. 最小栈

### 1. 题目链接
[https://leetcode.cn/problems/min-stack/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
只想到用一个变量实时刷新最小值，但 pop 之后就无法找回历史记录的前一个最小值了。
**易错点：** 在同步入栈操作时，最小栈由于可能遇到相同最小值，因此入栈辅助栈的判断条件必须是 `<=`，避免重复最小值在常规栈出栈时被误删。

### 3. 核心解法：辅助最小栈
**3.1 思路分析**
**破局点：** 利用一个**辅助栈**与常规栈同步进出，永远将当前层的历史最小值保存在上面。
- 常规栈正常压入弹出。
- 最小栈 push 小于等于当前栈顶的元素。
- 当常规栈弹出的正好是当前的最小元素时，最小栈同步剥离外层。

**3.2 代码实现 (C++)**
```C++ 
class MinStack {
    stack<int> min_stack;
    stack<int> norm_stack;
public:
    MinStack() {
        min_stack.push(INT_MAX); // 避免边界空栈判断
    }
    void push(int val) {
        norm_stack.push(val);
        if(val <= min_stack.top()){ 
            min_stack.push(val);
        }
    }
    void pop() {
        int top = norm_stack.top();
        norm_stack.pop();
        if(top == min_stack.top()){
            min_stack.pop();
        }
    }
    int top() {
        return norm_stack.top();
    }
    int getMin() {
        return min_stack.top();
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(1)$。所有基本入栈出栈及获取最值操作均基于栈顶。
- **空间复杂度**: $O(N)$。辅助最小栈同样随着元素规模变动。

### 5. 总结与反思
通过空间换时间同步存储“历史快照状态”满足 O(1) 提取最值的变态要求。
