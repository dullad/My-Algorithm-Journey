# LeetCode 20. 有效的括号

### 1. 题目链接
[https://leetcode.cn/problems/valid-parentheses/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
左括号一直入栈，一旦遇到右括号，开始处理匹配，但只想到顺序处理。
**易错点：** 奇数长度直接排除，最后遍历结束栈中可能还有未匹配的括号（比如都是左括号）。

### 3. 核心解法：栈匹配
**3.1 思路分析**
**破局点：** 利用栈的**后进先出**特性，遇到左括号入栈，遇到右括号则与栈顶匹配，匹配失败或栈空则无效。所有括号抵消完说明有效。
- 遇左括号入栈。
- 遇右括号弹栈比对匹配。
- 遍历结束，判断栈是否为空。

**3.2 代码实现 (C++)**
```C++
class Solution {
public:
    bool isValid(string s) {
        stack<char> myStack;
        int n = s.size();
        if(n % 2 != 0) return false;
        for(int i = 0; i < n; i++){
            if(s[i] == '(' || s[i] == '[' || s[i] == '{'){
                myStack.push(s[i]);
            }else{
                if(myStack.empty()) return false;// 第一个为右括号
                char left = myStack.top();
                if((left == '(' && s[i] == ')') || (left == '[' && s[i] == ']') || (left == '{' && s[i] == '}')){
                    myStack.pop();
                }else{
                    return false; // 不匹配
                }
            }
        }
        return myStack.empty();
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N)$。只需一次遍历字符串。
- **空间复杂度**: $O(N)$。最坏情况下所有字符均为左括号入栈。

### 5. 总结与反思
括号和边界匹配场景首选栈，巧用栈顶匹配迅速排重。
