# LeetCode 739. 每日温度

### 1. 题目链接
[https://leetcode.cn/problems/daily-temperatures/description/?envType=study-plan-v2&envId=top-100-liked]

### 2. 我的初次思路与遇到的问题 (The Error in "Error Log")
利用栈做逐个比对，但没准确理出该压入怎样的元素进行标记。
**易错点：** 求“需要几天后”，差值即下标距离。若栈中直接存储具体的数值而不是**索引序列下标**将无法轻松做差计日。

### 3. 核心解法：单调栈
**3.1 思路分析**
**破局点：** 构建一个**递减的单调栈记录位置**。如果当前元素一直萎靡走低，则下标默默排队入栈等出头日。一旦遇到比栈顶高的元素，代表它就是栈顶下标要苦苦寻找的“升温临界点”，弹出并利用下标做差即可。

**3.2 代码实现 (C++)**
```C++
class Solution {
public:
    vector<int> dailyTemperatures(vector<int>& temperatures) {
        stack<int> st;
        vector<int> res(temperatures.size(), 0);
        st.push(0);
        for(int i = 1; i < temperatures.size(); i++){
            if(temperatures[i] <= temperatures[st.top()]){
                st.push(i);
            }
            else{
                while(!st.empty() && temperatures[i] > temperatures[st.top()]){
                    res[st.top()] = i - st.top();
                    st.pop();
                }
                st.push(i);
            }
        }
        return res;
    }
};
```

### 4. 复杂度分析
- **时间复杂度**: $O(N)$。全体天数索引最多仅经历一次入栈与随后的弹栈。
- **空间复杂度**: $O(N)$。单调栈为了储存找不到比它高的数据极限规模。

### 5. 总结与反思
寻找数组中“最近的下一个比其最大或最小的元素位置”直觉锁定单调栈思想。
